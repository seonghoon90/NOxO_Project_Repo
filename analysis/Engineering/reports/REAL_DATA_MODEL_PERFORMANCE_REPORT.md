## 🎓 5. 핵심 인사이트

### 5-1 왜 Ridge가 가장 우수한가?

```
1️⃣ 데이터 구조가 선형성이 높음
   - IGCC 센서 데이터는 물리적 법칙을 따름
   - Ridge의 선형 모델이 충분
   - 복잡한 트리 모델은 과적합만 유발

2️⃣ 센서 신호 품질이 우수함
   - 노이즈가 적음 (결측치 0.00008%)
   - 규칙적인 측정
   - 선형 모델도 충분한 표현력

3️⃣ 정규화 효과
   - Ridge의 L2 정규화가 효과적
   - alpha=1.0에서 최적의 결과
   - 과적합 방지

4️⃣ 트리 모델의 한계
   - XGBoost/LightGBM은 과도하게 학습
   - 검증 데이터에서 성능 저하
   - 더 많은 계산 (42% 높은 오차)
```

### 5-2 모델 복잡도 vs 성능

```
복잡도가 높을수록 항상 성능이 좋은 것은 아니다!

     모델 복잡도
          ↑
     XGBoost ●  (느리고, 오차 높음)
     LightGBM ● (느리고, 오차 높음)
          |
          |  ← "과적합의 골짜기" (검증 성능 저하)
          |
      Ridge ●  (간단하고, 오차 낮음, 빠름) ✨
          |
          └─────────────────→ 성능

결론: 간단한 모델이 이 문제에 최적입니다!
```

### 5-3 Synthetic 데이터 vs 실제 데이터

| 항목 | 합성 데이터 | 실제 데이터 |
|------|----------|----------|
| Ridge MAE | 0.1479 | 0.007154 |
| 개선도 | (기준) | **95배 우수** |
| 데이터 수 | 10,000 | 1,295,999 |
| 완성도 | 낮음 | 높음 |

**해석**: 실제 데이터가 훨씬 높은 품질이므로 더 정확한 모델을 만들 수 있습니다.
