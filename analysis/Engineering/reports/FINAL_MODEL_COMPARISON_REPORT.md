## 💡 핵심 발견

### 1️⃣ 단순 모델의 강점

당신의 데이터셋에서는 **비선형 모델보다 선형 모델(Ridge)이 더 잘 작동**합니다.

**이유 분석:**
- NOx 예측 문제는 기본적으로 선형 구조일 가능성
- 트리 기반 모델들의 과적합 경향 (Train-Test gap)
- 피처 수가 충분해서 선형 모델도 충분한 표현력

### 2️⃣ 성능-속도 Trade-off

```
    속도
     ↑
     |    Ridge ⭐⭐⭐⭐⭐ (0.004초)
     |      |
     |    LightGBM ⭐⭐⭐⭐ (0.54초)
     |      |
     |    XGBoost ⭐⭐⭐⭐ (0.83초)
     |      |
     |    SVR (1.90초)
     |      |
     |    GradientBoosting (48.16초)
     |
     +─────────────────────────→ 성능 (MAE 낮을수록 ○)
     0.14            0.16
```

**결론**: Ridge는 "파레토 프론티어" 위에 있음 (속도와 성능 모두 최고)

### 3️⃣ 과적합 분석

**Train-Test MAE 차이 (작을수록 좋음)**
- Ridge: 0.004 (우수) ✅
- LightGBM: 0.011 (우수) ✅
- XGBoost: 0.053 (경고) ⚠️
- GradientBoosting: 0.049 (경고) ⚠️
- SVR: 0.151 (심각) ❌

→ **Ridge가 가장 안정적**

### 1️⃣ 하이퍼파라미터 튜닝

```python
# 시도: alpha in [0.001, 0.01, 0.1, 1.0, 10.0, 100.0]
```

**예상 개선**: 추가 0.5~1% 성능 향상 가능

### 2️⃣ Feature Engineering

새로운 파생 피처 생성:
- 온도 × 압력 상호작용
- 시간 윈도우 통계량
- 도메인 기반 지표

**예상 개선**: 추가 2~5% 가능

### 3️⃣ 앙상블

Ridge + LightGBM 가중 조합

**예상 개선**: 0.5~1% 가능
